CA Final Project

Project Topic
- LS Channel Estimation + LMMSE/MMSE Equalizer Acceleration

Implemented Parts
- Part 1: Scalar baseline
- Part 2: RVV vector reduction
- Part 3: SIMD-like RVV parallelization
- Part 4: CUDA SIMT single-pattern implementation
- Part 5: Multi-pattern GPU parallelism

Repository Structure
- common/      Shared parameters, data arrays, I/O, model, and verification helpers
- data_gen/    Single-pattern OFDM input generator
- data/        Generated binary input files
- part1/       Scalar baseline
- part2/       RVV reduction implementation
- part3/       SIMD-like RVV implementation
- part4/       CUDA SIMT implementation
- part5/       Multi-pattern CUDA implementation
- reference/   Assignment PDF, RVV spec, and tutor examples
- figures/     Report figures

Environment Requirements

Part 1 to Part 3
- Docker with the course gem5 / RVV image
- RISC-V cross compiler inside the container

Part 4 and Part 5
- NVIDIA GPU with CUDA support
- Docker with the course CUDA image

Input Generation
- Part 1 to Part 4 use data/ofdm_input.bin
- This file is generated by data_gen/generate_ofdm_data.cpp
- Part 5 uses data/ofdm_input_multi.bin
- This file is generated by part5/generate_ofdm_multi.cpp

How to Build and Run

Start from the repository root directory.

Part 1
- Host correctness check:
  make -C part1 clean
  make -C part1 host
- gem5 run:
  make -C part1 clean
  make -C part1

Part 2
- gem5 run:
  make -C part2 clean
  make -C part2

Part 3
- gem5 run:
  make -C part3 clean
  make -C part3

Notes for Part 1 to Part 3
- Each make run regenerates data/ofdm_input.bin on the host side
- The program is then cross-compiled and executed in gem5 TimingSimpleCPU
- Important statistics are written under:
  part1/m5out/
  part2/m5out/
  part3/m5out/

Part 4
- Build:
  make -C part4 clean
  make -C part4
- Default GPU run:
  make -C part4 run
- Optional commands:
  make -C part4 ptx
  make -C part4 profile
  make -C part4 sweep
- Results directory:
  part4/results/

Part 5
- Build:
  make -C part5 clean
  make -C part5
- Default GPU run:
  make -C part5 run
- CPU baseline:
  make -C part5 run_cpu
- Optional commands:
  make -C part5 ptx
  make -C part5 profile
  make -C part5 sweep
- Results directory:
  part5/results/

Output / Verification
- All parts print correctness-related metrics such as:
  H_MSE
  MSE_RX_BEFORE_EQ
  MSE_LMMSE
  checksum
  Verification: PASS / FAIL

Measurement Scope
- Part 1 to Part 3 use gem5 simulated statistics
- Part 4 and Part 5 use GPU kernel-only timing measured by cudaEvent
- GPU kernel-only timing does not include:
  input loading
  allocation / free
  host-device copies
  host-side verification

Final Report
- The final report file is Report.pdf
